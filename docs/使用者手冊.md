## 🌟 這個系統能幫你做什麼？

這是一套 **AI 自動化內容生產系統**，可以幫你：

| 功能 | 說明 |
|------|------|
| ✍️ **寫文章** | 自動撰寫符合 SEO 規範的部落格文章 |
| 🔍 **SEO 優化** | 自動加入 Meta Tags、FAQ Schema、關鍵字優化 |
| ✅ **事實查核** | 檢查文章中的數據和連結是否正確 |
| 🔗 **連結管理** | 自動加入內部連結和 CTA 按鈕 |
| 📤 **一鍵發布** | 直接發布到 WordPress 網站 |
| 📚 **舊文翻新** | 自動抓取舊文章並重新優化 |

## 🛠️ 環境準備

### 首次設置（只需做一次）

#### 步驟 1：安裝依賴套件

打開終端機（Terminal），輸入以下指令：

```bash
# 進入專案資料夾
cd "專案資料夾路徑"

# 建立虛擬環境
python3 -m venv venv

# 啟動虛擬環境
source venv/bin/activate

# 安裝套件
pip install -r requirements.txt
```

#### 步驟 2：設定環境變數

```bash
# 複製設定範本
cp .env.example .env
```

然後用文字編輯器打開 `.env` 檔案，填入以下資訊：

```
# WordPress 網站
WP_BASE_URL=你的網站網址
WP_USERNAME=WordPress帳號
WP_APPLICATION_PASSWORD=應用程式密碼

# Google API
GOOGLE_API_KEY=你的Google API金鑰
```

> 💡 **什麼是應用程式密碼？**
> 
> 這不是你的登入密碼！請到 WordPress 後台 → 使用者 → 個人資料 → 應用程式密碼 → 新增。

#### 步驟 3：建立向量資料庫

```bash
python agents/site_auditor.py
```

這個步驟會掃描你的網站，建立本地搜尋索引。首次執行約需 3-5 分鐘。

## ❓ 常見問題 FAQ

### Q: 我不知道要用什麼指令怎麼辦？

**A:** 直接用自然語言說出你想做的事！例如：
- 「我想寫一篇關於 XX 的文章」
- 「幫我優化這篇文章」
- 「把這篇發布到網站」

系統會自動選擇正確的流程。

---

### Q: 文章發布失敗怎麼辦？

**A:** 請依序檢查：

1. **確認 `.env` 設定正確**
   - 網址結尾不要有斜線 `/`
   - 應用程式密碼沒有空格

2. **測試 WordPress 連線**
   ```
   輸入：「檢查 WordPress 連線」
   ```

3. **重新產生應用程式密碼**
   - 到 WordPress 後台重新產生

---

### Q: 向量資料庫錯誤怎麼辦？

**A:** 重新建立資料庫即可：

```bash
# 刪除舊資料庫
rm -rf data/lancedb data/chromadb_*

# 重新掃描建立
python agents/site_auditor.py
```

---

### Q: 如何讓 AI 學習我的寫作風格？

**A:** 使用風格學習功能：

1. 先在 WordPress 發布一篇文章
2. 手動修改文章內容（加入你的風格）
3. 執行 `/34_風格學習`

系統會分析你的修改，學習你的語氣偏好。

---

### Q: 可以批次處理多篇文章嗎？

**A:** 可以！使用批次風格學習：

```
/批次風格學習
```

這會掃描所有被你編輯過的文章，批量學習你的風格。

## 📁 專案資料夾結構

### 內容產出流程與存放位置

當你執行內容生產流程時，檔案會依照以下順序存放：

```
outputs/FUNIT/
├── strategies/      ← 步驟 1：關鍵字策略 JSON
├── briefs/          ← 步驟 2：內容大綱 Brief
├── drafts/          ← 步驟 3：文章草稿 (Markdown)
├── optimized/       ← 步驟 4：SEO 優化後的文章
├── final/           ← 步驟 5：最終版本（準備發布）
├── reports/         ← 事實查核報告
├── logs/            ← 執行日誌
└── 收集到的資料/    ← 全站掃描資料
    ├── site_structure.json    ← 網站結構
    ├── posts_index.json       ← 文章索引
    └── categories_summary.md  ← 分類摘要
```

### 設定檔位置

| 檔案 | 用途 |
|------|------|
| `config/brand_profile.json` | 品牌基本設定（名稱、網址、語言） |
| `docs/brand_guideline.md` | 品牌語氣與風格指南 |
| `.env` | API 金鑰與 WordPress 連線設定 |

### 資料庫位置

| 資料夾 | 用途 |
|--------|------|
| `data/lancedb/` | 向量資料庫（全站內容索引） |
| `data/chromadb_style/` | 風格規則資料庫 |

## ⚠️ 資料夾命名限制

> **重要**：以下資料夾名稱是程式碼中硬編碼的，**請勿修改**！

### 🔒 不可修改

| 資料夾 | 說明 |
|--------|------|
| `outputs/` | 所有產出的存放位置 |
| `data/` | 向量資料庫存放位置 |
| `config/` | 系統與品牌設定檔 |
| `docs/` | 品牌指南與文件 |
| `agents/` | AI Agent 程式碼 |
| `.gemini/skills/` | Gemini Skills 定義檔 |

### ✅ 可以修改

| 項目 | 修改方式 |
|------|----------|
| 品牌名稱 (如 `FUNIT`) | 編輯 `config/brand_profile.json` 中的 `brand_identity.name` |

修改後系統會自動在 `outputs/` 下建立新的品牌資料夾。
